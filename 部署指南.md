# 飞机大战游戏部署指南

## 部署到GitHub Pages

### 1. 在GitHub上创建新仓库
1. 打开 [GitHub](https://github.com/) 并登录你的账号
2. 点击右上角的 "+", 选择 "New repository"
3. 填写仓库信息：
   - Repository name: 输入仓库名称，例如 "airplane-game"
   - Description: 可选，输入 "网页版飞机大战游戏"
   - 选择 "Public"（必须是公开仓库才能使用GitHub Pages）
   - 不要勾选 "Add a README file"（我们已经有了）
   - 点击 "Create repository"

### 2. 将本地仓库推送到GitHub

在本地仓库目录中执行以下命令：

```bash
# 添加远程仓库
# 替换 YOUR_GITHUB_USERNAME 为你的GitHub用户名，YOUR_REPOSITORY_NAME 为你创建的仓库名称
git remote add origin https://github.com/YOUR_GITHUB_USERNAME/YOUR_REPOSITORY_NAME.git

# 推送本地仓库到GitHub
git push -u origin master
```

### 3. 启用GitHub Pages
1. 打开你在GitHub上创建的仓库
2. 点击 "Settings"（设置）选项卡
3. 在左侧菜单中选择 "Pages"
4. 在 "Source" 部分：
   - 选择 "Deploy from a branch"
   - 在 "Branch" 下拉菜单中选择 "master"（或 "main"）分支
   - 在 "Folder" 下拉菜单中选择 "/ (root)"
   - 点击 "Save"

### 4. 获取外网链接
- 保存设置后，GitHub Pages会开始部署你的网站
- 部署完成后，你会在 "GitHub Pages" 设置页面看到类似 "Your site is live at https://YOUR_GITHUB_USERNAME.github.io/YOUR_REPOSITORY_NAME/" 的链接
- 这个链接就是你的游戏外网访问地址，复制并分享给他人即可

## 部署到其他平台

### Vercel部署
1. 访问 [Vercel](https://vercel.com/) 并登录
2. 点击 "Add New Project"
3. 选择 "Import Git Repository"
4. 连接你的GitHub账号，选择你的游戏仓库
5. 点击 "Deploy"，等待部署完成
6. 部署完成后，Vercel会生成一个外网链接

### Netlify部署
1. 访问 [Netlify](https://www.netlify.com/) 并登录
2. 点击 "Add new site" -> "Import an existing project"
3. 连接你的GitHub账号，选择你的游戏仓库
4. 点击 "Deploy site"
5. 部署完成后，Netlify会生成一个外网链接

## 注意事项

1. **部署时间**：GitHub Pages部署通常需要1-2分钟，Vercel和Netlify通常更快
2. **更新部署**：每次修改代码并推送到GitHub后，GitHub Pages会自动重新部署
3. **自定义域名**：如果需要使用自定义域名，可以在各平台的设置中配置
4. **HTTPS**：所有平台都默认提供HTTPS支持

## 测试外网链接

部署完成后，打开生成的外网链接，测试游戏是否能正常运行：
1. 检查游戏是否能正常加载
2. 测试WASD控制是否正常
3. 测试游戏功能是否完整
4. 测试在不同浏览器中的兼容性
